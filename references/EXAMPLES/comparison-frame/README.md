# comparison-frame —— 竞品对比独立章节（C3 anchor）

> ⚠️ **结构示意，不是抄袭模板**。看「形」，按你的内容 + 主题换内容选型。

## 适用场景

口播在讲**「我们 vs 同类」**：

- 功能对比表（谁支持 X / Y / Z）
- 方案选型（A / B / 我们各自的取舍）
- 「为什么不用现成的」反驳

凡是需要观众**一眼看出「只有我们全勾」**的时候用它。

## 核心手段（落地 Vizplainer #8 签名）

| 签名 | 这里怎么用 |
|---|---|
| **#8 竞品对比作为独立章节** | 不是正文里一句「比 X 好」，而是给它**一整屏一张对比表**。让对比有仪式感。 |
| **对照列灰态** | 竞品列整列 `--text-faint` / `--text-ghost`（退到背景），我方列 accent 描边 + `--accent-soft` 微底。视觉上「我们」立刻跳出来。 |
| **逐列揭示** | 对照列先一列列刷出 → 我方列最后压轴。**禁一次全亮**（那样观众没有「看着差距拉开」的过程）。每列 1 step。 |
| **✓ / ✕ 用字符不用 emoji** | `✓` / `✕` 是纯字符 + token 色（反 AI 味铁律：不用 emoji 当图标）。 |
| **#1 token 文字层级** | 用到了新的 `--text-ghost`（第 4 档）让对照列的「无」状态比「有」更淡，层级更细。 |

## step 切分

```
step 0  设问    表头 + 行标 + 一句问题（列还没出）
step 1  竞品1    Legacy A 列灰态刷出
step 2  竞品2    Cloud B 列灰态刷出
step 3  我们     我方列 accent 压轴刷出 —— 全勾，定胜负
```

→ narrations.ts **必须 N+1 条**（这里 3 列 = 4 条）。

## 用到的全局原语 / token

- `--text-ghost`（base.css，A1 新增的第 4 档文字色，用于「无」状态）
- `.sys-label`（行标 / 能力名）
- `--accent-soft` / `--accent`（我方列高亮）

## 切到其它主题怎么换

- **表格结构不动**。
- 暗色主题：我方列加 glow（`box-shadow: 0 0 40px var(--accent-glow)`）。
- 浅色主题：`--accent-soft` 自动变浅底，描边照常。
- bauhaus-bold：把我方列描边换成 4px 粗黑边 + accent 块，更布鲁塔。

## 反模式提醒

- ✗ 三列一次全亮 → 没有「差距拉开」的叙事，沦为静态表。**逐列揭示**才有戏。
- ✗ 竞品列也上色 → 视觉打架，「我们」不再跳出。**对照必须灰**。
- ✗ 用 ✅❌ emoji → 触发反 AI 味铁律。用字符 `✓` `✕` + token 色。
- ✗ 把对比埋进别的章节角落 → 失去 #8 的「独立仪式感」。给它整整一屏。
